# 開発運用ガイドライン（セッション/コミット）

本ドキュメントは、**セッションの運用ルール**と**自動コミット**の扱いを整理したものです。

## 1. セッション開始時のルール

- **必ず `docs/README.md` を参照**してから作業を開始する。
- 以降のやりとりは**日本語で統一**する（英語での回答は行わない）。

## 2. 自動コミットの運用ルール

### 2-1. トリガー
以下のような**肯定的な合図**が来たタイミングで、実装内容を自動コミットする。

- 「OK」「いい感じ」「この方針で進めて」「助かった」などの**肯定的プロンプト**

### 2-2. ステージ範囲

- **作業で変更したものはすべてステージ**してコミットする。
- 不要な変更が混ざっている場合は、必ず整理してからコミットする。

### 2-3. コミットメッセージ

- **日本語で記述**する。
- 例: `公式スポンサーセクションのレイアウト調整`

### 2-4. 実行コマンド

```
npm run commit:auto -- "コミットメッセージ"
```

## 3. 注意事項

- 自動コミットは**セッション内の承認タイミングに合わせて実行**する。
- 変更差分がない場合はコミットを行わない。
